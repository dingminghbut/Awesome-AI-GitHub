# 从这里开始

[返回 README](https://github.com/dingminghbut/Awesome-AI-GitHub/blob/main/README.zh-CN.md) | [在线榜单](https://dingminghbut.github.io/Awesome-AI-GitHub/) | [每周摘要](reports/weekly-digest.zh-CN.md) | [排名方法](METHODOLOGY.zh-CN.md)

Awesome AI GitHub 面向想快速发现 AI 开源项目的人：不用每天手动刷多个榜单，也能看到正在增长、值得进一步了解的项目。

## 按你的目标进入

- 做 LLM 或聊天应用：先看[大语言模型与聊天](https://dingminghbut.github.io/Awesome-AI-GitHub/llm.html)。
- 做智能体或自动化工作流：先看[智能体](https://dingminghbut.github.io/Awesome-AI-GitHub/agents.html)。
- 做知识库应用：先看[RAG 与知识库](https://dingminghbut.github.io/Awesome-AI-GitHub/rag.html)。
- 做图像、视频、音频或创意工具：先看[生成式 AI](https://dingminghbut.github.io/Awesome-AI-GitHub/generative-ai.html)。
- 做模型部署、推理或工程化：先看[AI 基础设施](https://dingminghbut.github.io/Awesome-AI-GitHub/ai-infra.html)。
- 关注模型质量对比：先看[评测与基准](https://dingminghbut.github.io/Awesome-AI-GitHub/evaluation.html)。
- 探索具身智能：先看[机器人与具身智能](https://dingminghbut.github.io/Awesome-AI-GitHub/robotics.html)。

## 怎么看标签

- 增长快：24 小时或 7 天增星明显。
- 近期活跃：仓库最近有 push。
- 有文档：存在 homepage、docs topic 或文档相关描述。
- 适合入门：出现 examples、tutorial、template、beginner 等信号。
- 偏生产可用：出现 deploy、serving、workflow、scale 等工程化信号，或项目活跃且基础热度较高。
- 研究/评测：出现 paper、benchmark、evaluation、leaderboard 等信号。
- 适合自托管：出现 local、private、offline、self-hosted 等信号。

## 十分钟使用流程

1. 打开[在线榜单](https://dingminghbut.github.io/Awesome-AI-GitHub/)，先看今日精选。
2. 按你当前工作方向选择分类。
3. 按你的场景看标签，不要只看总 stars。
4. 打开两三个仓库，检查 examples、docs、issues 和 release 活跃度。
5. 如果发现值得追踪的新项目，可以提交推荐 issue。
